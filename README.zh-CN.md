# 振动计数器

一个单文件手机网页工具,用设备的运动传感器(motion sensor)**实时检测并计数振动 / 敲击 / 冲击**——为重复性的物理计数而做(例如敲击计数菌落),在 **iPhone 或 Apple Watch** 浏览器里直接打开,免安装。

## 它能做什么

在手机(或 Apple Watch)上打开页面,授权运动传感器,点按大数字即可开始。每一次落在你设定强度区间内的物理振动都会让计数 +1。一个**高通滤波(high-pass filter,只保留快速变化、滤掉缓慢分量)**会持续把重力和朝向吸收进一条缓慢基线,因此只测量*偏离基线*的部分——也就是真正的抖动。你用三个旋钮(灵敏度、动作上限、冷却时间)把它调成只数你关心的那种敲击,既忽略背景抖动,也忽略过大的撞击。

它是一个**免手操作的物理事件计数器**,不是科学级加速度记录仪——每个数字都是从消费级运动 API(应用编程接口)读出的启发式估计,而非经过标定的测量值。

## 怎么用

1. **用 HTTPS 打开。** 运动传感器只在安全上下文(secure context)可用;请通过 `https://`(如 GitHub Pages)访问,而不是直接 `file://` 打开。
2. **授权。** iOS 13+ 会出现一个按钮——点**授权传感器**并在系统弹窗里同意。
3. **点按大数字开始。** 状态点变为激活,检测开始。
4. **再次点按数字归零**(并强制重新订阅传感器流);**长按(约 0.6 秒)暂停 / 恢复**,计数不丢失。
5. **看实时面板:** 加速度条显示当前强度及你的下限 / 上限标记;三个统计格分别是**峰值(m/s²)**、**速率(次/分钟)**、**计时**。
6. **快速换算:** 点 `×2 / ×4 / ×8 / ×63.6` 任一按钮,把 `当前计数 × 系数` 的快照显示在按钮上;再点一次还原为系数标签。

## 操作与显示

| 元素 | 动作 / 含义 |
| --- | --- |
| 大数字 | **点按** = 归零 · **长按** = 暂停 / 恢复 · 同时也是启动触发 |
| 加速度条 | 实时振动强度(0–30 m/s²),带下限与上限标记 |
| 峰值 / 速率 / 计时 | 本次会话峰值强度、每分钟次数、已用时长 |
| `×2 ×4 ×8 ×63.6` | 在按钮上切换 `计数 × 系数` 快照(系数为用户自定义,无固定物理含义) |

## 设置(灵敏度设置)

| 设置 | 范围 | 作用 |
| --- | --- | --- |
| **灵敏度** | 1–10 | 映射为下限阈值(1 ≈ 20 m/s²,需要大动作 → 10 ≈ 2 m/s²,轻微振动即触发)。 |
| **动作上限** | 20–45 | 强于此值的振动视为过大冲击,**不**计数;拉到顶端(45)= 不限。 |
| **冷却时间** | 100–1000 ms | 两次计数的最小间隔;持续振动按此节奏计数。 |
| **当前区间** | (显示) | 显示生效的 `下限 – 上限` 区间;只有落在区间内的振动才计数。 |

## 底层原理

- **高通滤波** —— 一条指数移动平均(EMA,α = 0.05)基线跟踪重力 / 朝向;计数用 `|合成幅值 − 基线|`,因此倾斜设备不会误触发。
- **触摸保护** —— 任意屏幕触摸后约 400 ms 内抑制计数,所以点按归零 / 暂停本身不会被当成一次振动。
- **多重防熄屏** —— Screen Wake Lock API + 一个近乎静音的 1 Hz AudioContext 振荡器 + MediaSession「正在播放」标记 + NoSleep.js,每一个都用 `try/catch` 包住,任一不支持就静默跳过。用来对抗 iOS / watchOS 自动锁屏。
- **挂起看门狗** —— iOS 在后台或空闲时会静默掐掉 `devicemotion` 事件流。一个 1 秒心跳,加上 `visibilitychange` / `pageshow` / 触摸钩子,在事件流静默超过 3 秒时自动重新订阅,让计数器自己「醒过来」。
- **rAF 合并 UI** —— 在 60 Hz+ 下,计数判断逻辑放在事件回调里实时执行以保证准确,而纯视觉更新被合并到每帧一次的 `requestAnimationFrame`,保持点击跟手。

## 环境要求与兼容性

- **必须 HTTPS / 安全上下文**才能访问传感器。
- **iPhone(iOS Safari)** —— 完整支持,含 iOS 13+ 权限弹窗与 Wake Lock。
- **Apple Watch(watchOS WebKit)** —— 目标平台;部分防熄屏手段(Wake Lock、NoSleep)可能不生效,但只要有运动事件到达,计数仍可用。
- **Android Chrome** —— 标准 `devicemotion` 路径可用;部分防熄屏技巧因厂商而异。
- 无需构建、无需安装依赖 —— 单个 `index.html`(NoSleep.js 从 CDN 加载,加载失败也不影响计数)。

## 局限

- **启发式估计,非标定值。** 数值来自消费级运动 API,不是计量级测量,且因设备而异。
- **基于阈值的计数。** 只有振动落在 `[下限, 上限]` 区间且过了冷却时间才计数——重叠或低于阈值的事件可能漏掉,冷却时间也限制了最高计数速率。
- **防熄屏只是尽力而为。** 不保证一定阻止自动锁屏,watchOS 上尤甚,屏幕仍可能休眠。
- **iOS 上事件流脆弱。** 尽管有看门狗,激进的后台切换仍可能中断事件流,直到你回到页面。
- **不持久化。** 计数、设置、统计都只在内存里;刷新即从零开始。

## 路线图 / 展望

- **持久化设置与会话**(localStorage),让灵敏度 / 冷却和上次计数在刷新后保留。
- **可配置的快算系数**,取代固定的 `×2/×4/×8/×63.6`。
- **导出 / 历史** —— 记录每次会话的计数、速率曲线与峰值供回看。
- **标定助手** —— 一套引导流程,为某种敲击方式挑选合适的灵敏度 / 阈值。
- **可安装为 PWA** —— 离线 manifest + service worker,像原生 App 一样启动。
